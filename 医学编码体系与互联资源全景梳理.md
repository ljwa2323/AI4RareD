# 医学编码体系与互联资源全景梳理（2026）

## 1. 目标与范围

本文聚焦两个问题：

1. 各类医学编码体系分别“管什么”；
2. ICD、OMIM、UMLS、SNOMED 及其他常见体系之间，能通过哪些权威资源实现连接（映射、对齐、跨库检索、语义桥接）。

说明：

- 本文优先采用官方机构站点与标准维护方信息；
- “连接”分为强连接（官方映射/可程序化 crosswalk）与弱连接（xrefs、同义词桥接、知识图谱对齐）；
- 不同资源的许可与用途限制差异很大，实施前需做合规审查。

## 2. 常见医学编码体系（按用途分组）

## 2.1 疾病与诊断分类

### ICD 家族（WHO）

- **ICD-11**：全球疾病分类与统计标准，原生数字化，提供 API；
- **ICD-10**：仍被大量国家地区用于统计和临床编码；
- **ICD-10-CM / ICD-10-PCS（美国）**：美国临床修订版与住院手术操作编码。

**示例数据（疾病：囊性纤维化 Cystic fibrosis）**


| 体系         | 编码        | 显示名称                                                                                     | 备注                      |
| ---------- | --------- | ---------------------------------------------------------------------------------------- | ----------------------- |
| ICD-11     | `5C58.02` | Cystic fibrosis                                                                          | WHO 数字化实体，可用 ICD API 查询 |
| ICD-10     | `E84`     | Cystic fibrosis                                                                          | 统计/病案常用三位码              |
| ICD-10-CM  | `E84.9`   | Cystic fibrosis, unspecified                                                             | 美国临床编码，比 ICD-10 更细      |
| ICD-10-PCS | `0B9D8ZX` | Drainage of Right Upper Lung Lobe, Via Natural/Artificial Opening Endoscopic, Diagnostic | 住院手术操作码，与诊断码分轴          |


FHIR 表示示例：

```json
{
  "system": "http://hl7.org/fhir/sid/icd-10-cm",
  "code": "E84.9",
  "display": "Cystic fibrosis, unspecified"
}
```

官方入口：

- WHO ICD 总入口：[https://www.who.int/standards/classifications/classification-of-diseases](https://www.who.int/standards/classifications/classification-of-diseases)
- ICD API：[https://icd.who.int/icdapi/](https://icd.who.int/icdapi/)
- CDC ICD-10-CM：[https://www.cdc.gov/nchs/icd/icd-10-cm/index.html](https://www.cdc.gov/nchs/icd/icd-10-cm/index.html)
- CMS ICD-10 资源：[https://www.cms.gov/medicare/coding-billing/icd-10-codes](https://www.cms.gov/medicare/coding-billing/icd-10-codes)

### SNOMED CT（SNOMED International）

- 细粒度临床术语系统，适合 EHR 结构化记录与语义计算；
- 常用于临床记录端，再映射到 ICD 等统计/报销分类端。

**示例数据**


| 编码 (conceptId) | 显示名称                                | 语义标签       |
| -------------- | ----------------------------------- | ---------- |
| `190905008`    | Cystic fibrosis (disorder)          | 疾病概念       |
| `44054006`     | Diabetes mellitus type 2 (disorder) | 疾病概念（常见对照） |
| `73211009`     | Diabetes mellitus (disorder)        | 更宽泛的父概念    |


FHIR 表示示例：

```json
{
  "system": "http://snomed.info/sct",
  "code": "190905008",
  "display": "Cystic fibrosis (disorder)"
}
```

官方入口：

- SNOMED CT Maps：[https://www.snomed.org/maps](https://www.snomed.org/maps)
- SNOMED CT to ICD-10 map 技术文档：[http://snomed.org/icd10map](http://snomed.org/icd10map)

### ORPHAcode / ORDO（Orphanet）

- 面向罕见病的专用术语与编码；
- 明确提供与 ICD-10、ICD-11、OMIM、SNOMED CT 的对齐关系（按映射精度分级）。

**示例数据**


| ORPHA 码 | 疾病名称            | 映射精度示例                                   |
| ------- | --------------- | ---------------------------------------- |
| `586`   | Cystic fibrosis | 与 ICD-10 `E84`、OMIM `219700` 为 exact 级对齐 |
| `324`   | Fabry disease   | 与 OMIM `301500` 对齐                       |
| `558`   | Marfan syndrome | 与 OMIM `154700` 对齐                       |


ORPHA 编码在 FHIR 中通常写作：

```json
{
  "system": "http://www.orpha.net",
  "code": "586",
  "display": "Cystic fibrosis"
}
```

官方入口：

- ORPHAcodes：[https://orphacodes.org/](https://orphacodes.org/)
- Mappings Browser：[https://mappings.orphacode.org/](https://mappings.orphacode.org/)

### OMIM（Johns Hopkins）

- 人类孟德尔遗传与基因-疾病知识库，MIM 编号是遗传病研究常用锚点；
- 提供 API（需 key，使用受协议约束）。

**示例数据**


| MIM 号    | 类型     | 名称                                     | 关联基因 |
| -------- | ------ | -------------------------------------- | ---- |
| `219700` | `#` 表型 | CYSTIC FIBROSIS                        | CFTR |
| `301500` | `#` 表型 | FABRY DISEASE                          | GLA  |
| `607854` | `*` 基因 | CF transmembrane conductance regulator | CFTR |


OMIM API 返回片段示例（`/entry/219700` 概念化）：

```json
{
  "omim": {
    "mimNumber": 219700,
    "titles": {
      "preferredTitle": "CYSTIC FIBROSIS; CF"
    },
    "geneMap": {
      "geneSymbols": "CFTR"
    }
  }
}
```

官方入口：

- API 说明：[https://www.omim.org/help/api](https://www.omim.org/help/api)
- API 申请：[https://omim.org/api](https://omim.org/api)

### MONDO（Monarch Initiative）

- 跨本体疾病整合层，目标是统一 OMIM、Orphanet、ICD、SNOMED 等多源疾病定义；
- 强调逻辑等价关系（不仅是松散 xref）。

**示例数据**


| MONDO ID        | 名称              | 聚合的外部 ID（节选）                                        |
| --------------- | --------------- | --------------------------------------------------- |
| `MONDO:0009061` | cystic fibrosis | OMIM:219700, Orphanet:586, DOID:1485, UMLS:C0010674 |
| `MONDO:0010526` | Fabry disease   | OMIM:301500, Orphanet:324                           |


OBO 格式一行示例：

```
[Term]
id: MONDO:0009061
name: cystic fibrosis
xref: OMIM:219700
xref: Orphanet:586
xref: DOID:1485
xref: UMLS:C0010674
```

官方入口：

- MONDO：[https://mondo.monarchinitiative.org/](https://mondo.monarchinitiative.org/)
- 文档：[https://mondo.readthedocs.io/en/stable](https://mondo.readthedocs.io/en/stable)

### Disease Ontology（DOID）

- 人类疾病本体，广泛用于科研数据整合；
- 长期维护对 ICD、SNOMED、OMIM、MeSH、UMLS 等 cross-reference。

**示例数据**


| DOID           | 名称              | 典型 xref                                                          |
| -------------- | --------------- | ---------------------------------------------------------------- |
| `DOID:1485`    | cystic fibrosis | ICD10CM:E84, SNOMEDCT:190905008, MESH:D003550, UMLS_CUI:C0010674 |
| `DOID:0050650` | Fabry disease   | OMIM:301500, Orphanet:324                                        |


官方入口：

- DO：[https://disease-ontology.org/](https://disease-ontology.org/)

### MedGen（NCBI）

- 医学遗传病与表型概念聚合门户，汇总 UMLS、OMIM、Orphanet、HPO、Mondo 等标识关系；
- 对罕见病/遗传病跨库检索非常实用。

**示例数据**

MedGen 概念页 `C0010674`（MedGen UID: 41393，Cystic fibrosis）聚合标识示例：


| 来源        | 标识                                      |
| --------- | --------------------------------------- |
| UMLS CUI  | `C0010674`                              |
| OMIM      | `219700`                                |
| Orphanet  | `586`                                   |
| MONDO     | `MONDO:0009061`                         |
| MeSH      | `D003550`                               |
| 关联 HPO 表型 | `HP:0002093`（Respiratory insufficiency） |


官方入口：

- MedGen：[https://www.ncbi.nlm.nih.gov/medgen/](https://www.ncbi.nlm.nih.gov/medgen/)
- FTP 映射文件目录：[https://ftp.ncbi.nlm.nih.gov/pub/medgen/](https://ftp.ncbi.nlm.nih.gov/pub/medgen/)

## 2.2 术语与元术语层

### UMLS（NLM）

- 不是单一编码，而是汇聚多术语源的“元术语系统”；
- 通过 CUI 把多词表概念对齐，是 ICD/MeSH/SNOMED/OMIM 等的关键桥梁；
- 关键文件：MRCONSO、MRREL、MRMAP、MRSTY 等。

**示例数据**

同一 CUI `C0010674` 在 MRCONSO 中对应多词表别名（节选）：


| SAB（来源词表）   | CODE        | STR（首选词）                     |
| ----------- | ----------- | ---------------------------- |
| SNOMEDCT_US | `190905008` | Cystic fibrosis (disorder)   |
| ICD10CM     | `E84.9`     | Cystic fibrosis, unspecified |
| MSH         | `D003550`   | Cystic Fibrosis              |
| OMIM        | `219700`    | CYSTIC FIBROSIS              |
| ORPHANET    | `586`       | Cystic fibrosis              |
| MTH         | —           | Cystic Fibrosis              |


MRMAP 映射关系示例（SNOMED -> ICD-10-CM，概念化）：


| MAPSET                 | FROM_ID     | TO_ID   | MAPRULE                     |
| ---------------------- | ----------- | ------- | --------------------------- |
| SNOMEDCT_US_to_ICD10CM | `190905008` | `E84.9` | IF 190905008 | ALWAYS E84.9 |


官方入口：

- UMLS 概览：[https://www.nlm.nih.gov/research/umls/about_umls.html](https://www.nlm.nih.gov/research/umls/about_umls.html)
- UTS API：[https://documentation.uts.nlm.nih.gov/](https://documentation.uts.nlm.nih.gov/)

### MeSH（NLM）

- 生物医学主题词表，主要用于文献索引与检索（PubMed 等）；
- 通过 UMLS、DO、MONDO、MedGen、OxO 可与疾病编码体系发生连接。

**示例数据**


| MeSH ID   | 首选主题词           | 树状号                 | PubMed 检索示例                     |
| --------- | --------------- | ------------------- | ------------------------------- |
| `D003550` | Cystic Fibrosis | C06.452.829.100     | `"Cystic Fibrosis"[MeSH Terms]` |
| `D005776` | Gaucher Disease | C16.320.565.925.249 | 罕见病文献检索锚点                       |


MeSH RDF URI 示例：`http://id.nlm.nih.gov/mesh/D003550`

官方入口：

- MeSH 首页：[https://www.nlm.nih.gov/mesh/meshhome.html](https://www.nlm.nih.gov/mesh/meshhome.html)
- MeSH Browser：[https://meshb.nlm.nih.gov/](https://meshb.nlm.nih.gov/)
- MeSH RDF/SPARQL：[https://id.nlm.nih.gov/mesh/](https://id.nlm.nih.gov/mesh/)

### NCIt / NCIm（NCI EVS）

- 癌症领域核心术语体系；
- 提供多术语映射（含 ICD、SNOMED、MedDRA 等场景），在肿瘤研究互操作中很常见。

**示例数据**


| NCIt Code | 名称                  | 典型映射                                   |
| --------- | ------------------- | -------------------------------------- |
| `C4872`   | Lung Adenocarcinoma | ICD-O-3 形态 `8140/3`，SNOMED `254637007` |
| `C9255`   | Cystic Fibrosis     | 与 OMIM `219700` 有 xref                 |


官方入口：

- NCI Vocabulary：[https://www.cancer.gov/about-nci/organization/cbiit/vocabulary](https://www.cancer.gov/about-nci/organization/cbiit/vocabulary)
- ICD-O-3.1 与 NCIt 映射示例：[https://evs.nci.nih.gov/ftp1/NCI_Thesaurus/Mappings/ICD-O-3_Mappings/About.html](https://evs.nci.nih.gov/ftp1/NCI_Thesaurus/Mappings/ICD-O-3_Mappings/About.html)

## 2.3 检验、药物、给药与单位

### LOINC（Regenstrief）

- 检验与观察项目编码标准；
- 与 SNOMED CT 形成联合互操作方案（LOINC Ontology）。

**示例数据**


| LOINC    | 全称                                          | 常用单位 (UCUM) | 临床场景  |
| -------- | ------------------------------------------- | ----------- | ----- |
| `2160-0` | Creatinine [Mass/volume] in Serum or Plasma | `mg/dL`     | 肾功能   |
| `4548-4` | Hemoglobin A1c/Hemoglobin.total in Blood    | `%`         | 糖尿病监测 |
| `8310-5` | Body temperature                            | `Cel`       | 生命体征  |


FHIR Observation 片段：

```json
{
  "code": {
    "coding": [{ "system": "http://loinc.org", "code": "2160-0", "display": "Creatinine [Mass/volume] in Serum or Plasma" }]
  },
  "valueQuantity": { "value": 1.1, "unit": "mg/dL", "system": "http://unitsofmeasure.org", "code": "mg/dL" }
}
```

官方入口：

- LOINC：[https://loinc.org/](https://loinc.org/)
- LOINC/SNOMED 协作站点：[https://loincsnomed.org/](https://loincsnomed.org/)

### RxNorm（NLM）

- 药品标准名与药物术语互操作枢纽（美国场景尤其关键）；
- 与 NDC 等可建立映射关系，可经 UMLS/OMOP 再跨到其他体系。

**示例数据**


| RxCUI    | 名称                               | Term Type | 说明             |
| -------- | -------------------------------- | --------- | -------------- |
| `313782` | acetaminophen 325 MG Oral Tablet | SCD       | 临床药物概念         |
| `161`    | acetaminophen                    | IN        | 成分（Ingredient） |
| `198440` | azithromycin 250 MG Oral Tablet  | SCD       | 抗生素示例          |


RxNorm API 关系示例（`/rxcui/313782/related?tty=NDC` 概念化）：

```json
{
  "relatedGroup": {
    "conceptGroup": [{
      "tty": "NDC",
      "conceptProperties": [{ "rxcui": "...", "name": "50580-506-01" }]
    }]
  }
}
```

官方入口：

- RxNorm 概览：[https://nlm.nih.gov/research/umls/rxnorm/overview.html](https://nlm.nih.gov/research/umls/rxnorm/overview.html)
- RxNorm API：[https://lhncbc.nlm.nih.gov/RxNav/APIs/RxNormAPIs.html](https://lhncbc.nlm.nih.gov/RxNav/APIs/RxNormAPIs.html)

### NDC（FDA）

- 美国药品国家编码（产品层）；
- openFDA 提供 API，可用于药品标识检索与联动。

**示例数据**


| NDC             | 产品                            | 剂型     | 关联 RxCUI               |
| --------------- | ----------------------------- | ------ | ---------------------- |
| `50580-506-01`  | Tylenol Extra Strength 500 mg | Tablet | 经 RxNorm 映射到对乙酰氨基酚产品概念 |
| `00069-0280-30` | Lipitor 10 mg                 | Tablet | 阿托伐他汀产品层标识             |


NDC 三段式结构：`labeler-product-package`（如 `50580-506-01`）

官方入口：

- openFDA Drug NDC：[https://open.fda.gov/apis/drug/ndc/](https://open.fda.gov/apis/drug/ndc/)

### ATC/DDD（WHOCC）

- 药物利用研究与统计监测常用分类；
- 常与 RxNorm/NDC/本地药品目录通过中间层做二次映射。

**示例数据**


| ATC 码     | 名称           | DDD       | 说明             |
| --------- | ------------ | --------- | -------------- |
| `N02BE01` | paracetamol  | 3 g (O)   | 镇痛/antipyretic |
| `J01FA09` | azithromycin | 0.3 g (O) | 大环内酯类抗生素       |
| `A10BA02` | metformin    | 2 g (O)   | 降糖药            |


ATC 五级结构示例：`N02BE01` = N( nervous ) > N02( analgesics ) > N02B( non-opioids ) > N02BE( anilides ) > N02BE01( paracetamol )

官方入口：

- WHOCC ATC/DDD：[https://atcddd.fhi.no/](https://atcddd.fhi.no/)

### UCUM（Units of Measure）

- 计量单位编码标准（检验结果互操作关键组件）；
- 常与 LOINC 一起使用，保障“项目 + 单位”的机器可解释性。

**示例数据**


| UCUM 码    | 含义    | 常见 LOINC 搭配             |
| --------- | ----- | ----------------------- |
| `mg/dL`   | 毫克每分升 | LOINC `2160-0`（肌酐）      |
| `mmol/L`  | 毫摩尔每升 | LOINC `2345-7`（葡萄糖）     |
| `Cel`     | 摄氏度   | LOINC `8310-5`（体温）      |
| `{ratio}` | 无量纲比值 | LOINC `4548-4`（HbA1c %） |


官方入口：

- UCUM 规范：[https://ucum.org/ucum.html](https://ucum.org/ucum.html)

## 2.4 操作、收费与监管

### CPT（AMA）

- 医疗服务与操作术语编码（美国）；
- 与 HCPCS Level I 对应，临床计费场景常见。

**示例数据**


| CPT     | 描述                                                           | 场景               |
| ------- | ------------------------------------------------------------ | ---------------- |
| `99213` | Office/outpatient visit, established patient, low complexity | 门诊复诊             |
| `71046` | Radiologic exam, chest, 2 views                              | 胸部 X 光           |
| `81479` | Unlisted molecular pathology procedure                       | 分子检测（含罕见病基因检测场景） |


官方入口：

- AMA CPT：[https://www.ama-assn.org/practice-management/cpt](https://www.ama-assn.org/practice-management/cpt)

### HCPCS（CMS）

- 美国医保理赔核心编码体系，分 Level I（CPT）和 Level II（耗材、设备、救护等）；
- 常与 ICD、CPT、NDC 等在支付流程中联动。

**示例数据**


| HCPCS   | 类型              | 描述                                                |
| ------- | --------------- | ------------------------------------------------- |
| `99213` | Level I (= CPT) | 门诊复诊                                              |
| `J0135` | Level II        | Injection, adalimumab, 20 mg                      |
| `E0601` | Level II        | Continuous positive airway pressure (CPAP) device |


支付链路示例：诊断 `E84.9` + 操作 `99213` + 药物 `J0135` 在同一理赔单中共现。

官方入口：

- CMS HCPCS：[https://www.cms.gov/medicare/coding/MedHCPCSGenInfo/Index.html](https://www.cms.gov/medicare/coding/MedHCPCSGenInfo/Index.html)

### MedDRA（ICH/MSSO）

- 药物警戒与不良事件监管编码标准；
- 与 SNOMED、ICD、NCIt 等通常通过项目级映射或中间层本体桥接。

**示例数据**

MedDRA 五级层级示例（术语：Headache）：


| 层级   | 编码         | 名称                       |
| ---- | ---------- | ------------------------ |
| SOC  | `10029205` | Nervous system disorders |
| HLGT | `10019231` | Headaches                |
| HLT  | `10019233` | Headaches NEC            |
| PT   | `10019211` | Headache                 |
| LLT  | `10019211` | Headache                 |


监管上报通常使用 PT 层编码 `10019211`。

官方入口：

- ICH MedDRA：[https://ich.org/page/meddra](https://ich.org/page/meddra)
- MSSO：[https://www.meddra.org/](https://www.meddra.org/)

## 2.5 功能、残疾与肿瘤专用补充分类

### ICF（WHO）

- 功能、残疾与健康状态分类；
- 与 ICD、ICHI 同属 WHO-FIC 框架，可作为功能维度补充。

**示例数据**


| ICF 码  | 名称                                               | 与 ICD 联用示例                                                         |
| ------ | ------------------------------------------------ | ------------------------------------------------------------------ |
| `b110` | Seeing functions                                 | 患者 ICD `H54.0`（ blindness ）+ ICF `b210`（ Hearing functions ）描述功能影响 |
| `d450` | Walking                                          | 囊性纤维化患者 `E84.9` + ICF `b440`/`d450` 描述呼吸/行走受限                      |
| `e110` | Products and technology for personal consumption | 辅助设备维度                                                             |


官方入口：

- WHO ICF：[https://www.who.int/standards/classifications/international-classification-of-functioning-disability-and-health](https://www.who.int/standards/classifications/international-classification-of-functioning-disability-and-health)

### ICD-O-3（WHO/IARC）

- 肿瘤登记常用多轴编码（部位 Topography + 形态 Morphology + 行为）；
- 与 ICD、SNOMED、NCIt 等在肿瘤数据集成中高频联动。

**示例数据**

肺腺癌登记记录示例：


| 轴              | 编码          | 含义                                                 |
| -------------- | ----------- | -------------------------------------------------- |
| Topography（部位） | `C34.1`     | Upper lobe, lung                                   |
| Morphology（形态） | `8140/3`    | Adenocarcinoma, NOS, malignant                     |
| Behavior（行为）   | `/3`        | Malignant（含在形态码第三位）                                |
| 对应 ICD-10      | `C34.1`     | Malignant neoplasm of upper lobe, bronchus or lung |
| 对应 SNOMED      | `254637007` | Adenocarcinoma of lung (disorder)                  |
| 对应 NCIt        | `C4872`     | Lung Adenocarcinoma                                |


官方入口：

- WHO ICD-O-3：[https://who.int/standards/classifications/other-classifications/international-classification-of-diseases-for-oncology](https://who.int/standards/classifications/other-classifications/international-classification-of-diseases-for-oncology)

## 4. ICD / OMIM / UMLS / SNOMED 重点连接路径（你关心的主线）

> 以下每条路径均给出**可落地的示例数据**，便于理解“输入什么码、经过哪层、输出什么码”。

## 4.1 主干路径 A：SNOMED -> ICD

**场景**：EHR 中记录 SNOMED 诊断，需生成 ICD 用于统计报送。


| 步骤  | 输入                                  | 连接资源                        | 输出                | 映射类型  |
| --- | ----------------------------------- | --------------------------- | ----------------- | ----- |
| 1   | SNOMED `190905008`（Cystic fibrosis） | SNOMED -> ICD-10 官方 Map     | ICD-10 `E84`      | exact |
| 2   | SNOMED `190905008`                  | NLM SNOMED -> ICD-10-CM Map | ICD-10-CM `E84.9` | exact |
| 3   | SNOMED `44054006`（DM type 2）        | 同上                          | ICD-10-CM `E11.9` | exact |


SNOMED Map 规则示例（概念化）：

```
IF 190905008 | Cystic fibrosis (disorder)
  ALWAYS E84.9 | Cystic fibrosis, unspecified
  MAP ADVICE: "Use additional code to identify any manifestations"
```

FHIR `$translate` 请求/响应示例：

```json
// POST [base]/ConceptMap/$translate
{
  "resourceType": "Parameters",
  "parameter": [
    { "name": "code", "valueCode": "190905008" },
    { "name": "system", "valueUri": "http://snomed.info/sct" },
    { "name": "targetsystem", "valueUri": "http://hl7.org/fhir/sid/icd-10-cm" }
  ]
}
// Response -> code: E84.9, display: Cystic fibrosis, unspecified
```

适用：EHR 到统计报表、医保结算、病案质控。

## 4.2 主干路径 B：OMIM -> UMLS -> SNOMED/ICD/MeSH

**场景**：遗传病研究数据库只有 OMIM 号，需扩展到临床/文献编码。


| 步骤  | 输入             | 连接资源                   | 输出                 |
| --- | -------------- | ---------------------- | ------------------ |
| 1   | OMIM `219700`  | UMLS MRCONSO（SAB=OMIM） | CUI `C0010674`     |
| 2   | CUI `C0010674` | UMLS MRCONSO 反查        | SNOMED `190905008` |
| 3   | CUI `C0010674` | UMLS MRCONSO 反查        | ICD-10-CM `E84.9`  |
| 4   | CUI `C0010674` | UMLS MRCONSO 反查        | MeSH `D003550`     |


UTS API 查询示例（概念化）：

```
GET /search/current?string=219700&sabs=OMIM
-> CUI: C0010674

GET /content/current/CUI/C0010674/atoms
-> SNOMEDCT_US: 190905008
-> ICD10CM: E84.9
-> MSH: D003550
-> ORPHANET: 586
```

完整链路一行表示：

```
OMIM:219700 --[CUI:C0010674]--> SNOMED:190905008 | ICD10CM:E84.9 | MESH:D003550 | ORPHANET:586
```

适用：遗传病知识入库、跨源证据汇总、罕见病 NLP 标准化。

## 4.3 主干路径 C：OMIM/ORPHA -> MONDO/MedGen -> 多体系

## 5. 实施建议：从“可用”到“可审计”

## 5.1 先定“主码系”，再做桥接

- 疾病域建议优先明确主码（例如 SNOMED 或 ICD），其他体系作为映射层；
- 避免同一业务字段混放多个主标准导致不可追溯。

## 5.2 映射必须带“证据等级”

- 建议在映射表中记录：来源（官方/第三方/人工）、版本号、生效日期、映射关系类型（exact/broad/narrow/related）；
- 对弱映射（尤其多跳 xref）必须设置人工复核流程。

## 5.3 版本治理不可省

- ICD、SNOMED、LOINC、RxNorm、OMOP vocab 都是持续更新的；
- 生产环境至少需要“版本冻结 + 升版回归 + 历史兼容策略”。

## 5.4 许可合规前置

- SNOMED、OMIM、CPT、MedDRA、部分 UMLS 源词表存在许可要求；
- 需要提前确认下载、再分发、商用嵌入、API 调用频率等条款。

---

## 6. 快速落地清单（按优先级）

1. 先做一个“疾病域最小可用桥接”：`ICD-10-CM + SNOMED + UMLS CUI + OMIM + ORPHA`；
2. 罕见病项目优先接入 `ORPHAcodes Mappings + MedGen_HPO_OMIM_Mapping`；
3. 检验与药物域补齐 `LOINC + UCUM + RxNorm (+NDC)`；
4. 如果要多中心研究，尽早引入 `OMOP Athena` 作为统一语义层；
5. 对外接口用 `FHIR ConceptMap/$translate` 暴露映射服务，便于系统集成。
