# 罕见病诊断技术发展阶段梳理（关键技术与代表论文）

## 1. 统计学与规则系统阶段（约 2008-2020）

### 1.1 阶段目标

建立可计算、可解释的表型表示与诊断评分体系，完成从病例表型到候选疾病的检索和排序。

### 1.2 关键技术

- **表型本体与语义相似度**：以 HPO 统一表型术语，支持病例与疾病知识库的语义匹配。
- **统计显著性辅助鉴别**：对候选诊断给出显著性提示，辅助专家判断。
- **似然比（LR）与后验概率**：融合表型和基因组证据，输出可解释的概率化排序结果。
- **权威知识库与术语映射**：Orphanet 提供命名、分类、跨术语映射，提升互操作性。

## 2. 传统机器学习阶段（约 2014-2021）

### 2.1 阶段目标

将诊断任务从“仅表型检索”推进到“表型 + 基因型联合推断”，以候选基因/变异排序提升临床效率。

### 2.2 关键技术

- **多源知识融合与本体传播**：跨本体传播隐含知识，对候选变异结果再排序。
- **网络传播与随机游走**：结合蛋白互作网络、跨物种表型与临床信息进行优先级排序。
- **文献驱动机器学习**：自动完成文献检索与病例知识匹配，加速孟德尔病诊断。
- **基因邻域推断**：利用同源、旁系、通路、互作邻居改进低排名致病基因的发现。
- **临床解释标准化**：ACMG/AMP 指南将变异证据组合规则标准化。

## 3. 深度学习阶段（约 2019-2025）

### 3.1 阶段目标

从结构化特征工程扩展到原始高维信号建模，包括面部图像、EHR 文本和变异功能效应。

### 3.2 关键技术

- **面部表型深度表征**：以深度模型进行遗传综合征 Top-k 识别。
- **EHR 表型抽取**：从自由文本自动抽取 HPO 术语并形成结构化表型向量。
- **EHR 人群早筛模型**：从超大规模 EHR 中识别疑似罕见病高风险个体。
- **变异功能效应深度预测**：例如剪接/错义影响预测，为变异解释提供 in silico 证据。

## 4. 大模型（LLM）阶段（约 2024-2026）

### 4.1 阶段目标

将罕见病诊断入口扩展到自然语言交互，支持鉴别诊断生成、证据整合与跨语言评估。

### 4.2 关键技术

- **零样本/少样本诊断生成**：根据病例摘要直接生成候选鉴别诊断。
- **RAG 检索增强**：通过知识库检索降低幻觉并增强可追溯性。
- **跨语言模板化评测**：基于 HPO 翻译和标准化病例模板进行多语言比较。
- **本体对齐自动评估**：利用 Mondo 等本体做同义词/层级映射，提升评测一致性。

## 5. Agent 与多模态系统阶段（约 2025-2026）

### 5.1 阶段目标

从“单模型输出”升级为“多工具协作流程”，实现可追溯证据链、多模态输入和自反思迭代。

### 5.2 关键技术

- **多智能体分层架构**：中心代理负责任务规划，专用代理调用工具完成子任务。
- **可追溯证据链推理**：每步推理与可验证证据绑定，强化临床可审计性。
- **知识图谱 + RAG 结构化检索**：面向特定场景（如面部表型）增强诊断一致性。
- **多源融合与强化学习组件**：融合 LLM、表型匹配等多策略以提升 Top-k 召回。
- **多模态输入融合**：联合文本、HPO、基因检测结果进行综合诊断。

## 6. 跨阶段观察（简要）

- **技术主线**：从“可解释规则”到“多源学习”，再到“自然语言交互”，最终走向“可审计的多代理流程”。
- **能力重心变化**：单点预测准确率逐步让位于端到端流程效率、证据可追溯性和临床协作能力。
- **现实结论**：在严格可复现基准上，传统工具仍具竞争力；LLM/Agent 的优势更多体现在交互、整合和流程加速。
